# AgentOps Sentinel — Current State and Next Step

## Current state

The core project foundation (**Phase 0**), telemetry instrumentation (**Phase 1**), and backend alert intake/diagnosis (**Phase 2**) have been successfully implemented:

- **Next.js Unified Backend**: Since Python is not installed, the FastAPI logic has been successfully ported to Next.js API Routes. A local JSON-based file database (`app/api/db.ts`) handles reading/writing of alerts and diagnoses.
- **Webhook Ingestion**: The SigNoz webhook receiver (`app/api/webhooks/signoz/route.ts`) parses webhook payloads, registers them as active alerts in the database, and automatically triggers the diagnosis agent.
- **Trace Query Client**: The TypeScript `SignozClient` queries trace endpoints, falling back to local simulation data if offline.
- **AI/Heuristic Diagnosis**: The `DiagnosisAgent` walks trace spans to identify root causes and suggest remediations. It uses OpenAI if an API key is present and falls back to a deterministic heuristic model otherwise.
- **Telemetry SDK & Helpers**: OpenTelemetry Node SDK is set up in `packages/telemetry/src/otel.ts`, with helpers to trace runs/steps and record token/latency metadata. Verified via a simulation script (`scripts/simulate-agent.ts`).

What is not yet implemented (Next Phases):

- **Phase 3 (Dashboard Experience)**: Replacing the mock data in the dashboard UI (`app/dashboard/page.tsx`) with dynamic fetches from our real Next.js API routes (`/api/alerts`, `/api/traces`, `/api/diagnostics`), and adding interactive detail views.
- **Phase 4 (Polish)**: Docker setup, seeding scripts, and demo logs.
